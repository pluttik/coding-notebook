Getting started

working directory

staging area aka index

repository

HEAD is the latest snapshot


Basic workflow

git init

git add

git commit -m "my new code does so-and-so"


Check process

git status //show changes in staging area relative to HEAD

git log //show past commits relative to HEAD

git checkout my_commit_number //go back to commit number my_commit_number

git checkout master //go back to HEAD

git reset --hard my_commit_number //go back to my_commit_number and lose everything after


Branching

git branch my_new_branch //create branch, gets its own HEAD

git checkout my_new_branch //switch to branch

git checkout -b my_new_branch //create and switch to branch

git checkout master //move to master for merging
git merge my_new_branch //merge new branch into master

git branch -d my_new_branch //delete the label to branch my_new_branch (commits remain)


merge conflicts //git checks for conflicts per line: one, the other or a combination


Remote repository

Main commands

git pull //with merge of your local branch (your working area)

git fetch //no merge yet

git push


Basic workflow

make new feature on new clone:

clone > branch > checkout branch > local commits > pull > resolve merge conflicts > push

while working on feature, stay up to date with remote:

checkout branch > git merge master

make new feature on existing repository:

pull > branch > checkout branch > add > commit > fetch master > checkout master > merge branch > resolve merge conflicts > test+fix > push

go back to earlier commit in a branch:

checkout commit_nr > branch new_branch


Extended workflow

[developer] make new feature:

1) start a new branch git checkout -b new_feature_branch

2) build new feature

3) add files to commit (while still on branch) git add .

4) commit changes (while still on branch) git commit -m "message"

5) push new branch to remote git push -u origin new_feature_branch

repeat 2 - 5 until new feature is ready for review

6) file a merge request on Gitlab for new_feature_branch

7) assign someone to review your code


[code reviewer] review new code:

1) create and checkout remote branch locally git checkout -b new_feature_branch origin/new_feature_branch

2) review the new code locally, e.g. by running unit tests

3) if OK, continue, if not, give feedback to [developer]

4) checkout master git checkout master

5) merge with master git merge new_feature_branch

6) fix any merge conflicts

7) push to origin/master git push
